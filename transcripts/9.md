# Day 9 — Prompting Fundamentals

Hi everyone, welcome back. This is Sierra Literacy Month. We'll continue to break down different concepts related to Sierra. Until now, it has been more academic in nature — focusing on certain overarching concepts and principles and key pieces of jargon related to artificial intelligence. Starting today and moving forward, I'd like this to become more and more practical, with more emphasis placed on actually how to use this, how to best leverage Sierra.

So we'll start with prompting. The key idea here that I hope to convey as it relates to prompting — or what you could think of as simply the instructions that we're providing for this machine — is that of specificity. If I give it really vague instruction — "do this," "write a report" — it's going to produce equally vague output. So this is arguably the single biggest lever that we have.

And there are a few key components related to specificity that I hope you incorporate into the instructions you're providing. Beginning with that of role — this is when we say "you are a technical reviewer, you need to summarize this and focus on that and produce this." The idea of giving it a role is really just guiding it, giving it an angle of approach — or you can think of it almost as the general perspective from which to approach the problem that we're working on.

Next is that of task. Again, this is the more explicit instruction we're providing — "summarize this, focus on this area and this section with special emphasis placed on this component," et cetera. The more explicit, the more detailed, the better. Often times, in addition to the instructions I provide, I like to provide a concrete example — "this is what I want you to do, and here's something to reference where I've actually done it, and this is how I need you to approach it or produce it as such."

Third is that of context. Now I know a lot of time has already been spent talking about context — this is the information that we provide. This includes the actual instruction we're providing, but I like to think of context in a lot of ways as simply the background information. Even if I'm speaking with a colleague on a specific problem, often times I can't really lead with the exact problem when I'm asking for some input or assistance. I need to give the backstory — "this is for this client, this is for this project, it's in this domain, these are some of the things I've already considered. Now that you have all this in context, let's talk about how to solve this specific problem."

And last is that of format. Again, this is just adding specification to the output that I'm looking for. I'm sure we've all seen that these machines run verbose by nature. If I don't give it some kind of guardrail or constraint in terms of the output I want, often times it's going to end up providing me with a lot of fluff and filler information that I just don't need. So I do want to specify — "I want a bulleted list, I want exactly 3 sentences, I want this in one page, I need this in a table with these columns," et cetera.

So lastly, I want to draw our attention to a few points at the bottom just to keep in mind as we start prompting and giving more instruction to these machines moving forward.

First is drawing our attention to the iterative nature of the way in which we communicate and work with this system, as opposed to Google. If I type in a query, it brings something back, and the conversation essentially stops there — based off of that initial query and what it's retrieved. The main difference, and really what makes AI systems so powerful, is the fact that I can then take its output and provide further instruction, and we can go back and forth as many times as necessary until I get exactly what I want — by providing more information, more context, more explicit instruction, more examples, et cetera. There's flexibility, and the conversation doesn't stop based off of what it's returned.

Next is that when I find a thread or a particular conversation just not really going anywhere — like it's gotten hung up on one key piece of information that was previously introduced and just doesn't seem to be reasoning in the direction I want — often times it's easier just to start a new thread entirely and start from scratch.

Lastly, and this is arguably the highest-impact approach to prompting: just ask Sierra itself to write the prompts for you. You can do this through a stream of consciousness. Often times what I do is just turn the microphone on within Sierra and start speaking — rough, not really instructed, not precise language. "Hey, this is what I was working on, this is the background, this is what I'm trying to do — based on all I've provided here, could you help me structure this into a more robust prompt that I can use moving forward?" And that is actually how the majority of the prompts I write are generated — from Sierra itself — based on a number of conversations and input sources I've provided, to give it all the relevant context and information to structure out a more robust prompt I can use moving forward.

So in the next, we'll start talking more about guiding the model through more complex tasks — really just structuring its workflow a little bit better, all for the purpose of getting better results and more accurate answers.

As always, really appreciate your time and attention — until next time.
