Hi everyone. Welcome back to Sierra Literacy Month.

In the last one I gave an overview of workspaces — introduced the idea of storing certain pieces of context as well as providing access to certain files within a more collaborative environment. So you can think of that as being set at more of the workspace or system level, where each chat within that workspace can then pull from this common set of context and files. And as we work through each chat, we can always be dynamically adding memories — stored pieces of information that will be included in context with each of our calls to the model.

In this one I want to focus more on what are called task modes. These are more dialed-in prompts that we can configure and save within a workspace. You can think of them simply as reusable step-by-step instructions, often for more specific, repeatable workflows. This could be a robust set of instructions — steps to take, which files it needs to reference, which general tools to activate, and then specifications around the output format.

As a general rule of thumb: any background information — this is the project, this is the client, this is our team, this is our scope — that is what you'd want to build out within the context of the workspace. All the background information that grounds the model and gives it a perspective or reference for approaching problems within the workspace. More specific, actionable, repeatable tasks — those are task modes. And again, these are simply prompts that we configure and save, then make available to anyone within that workspace in any chat.

We'll of course be covering a number of different examples. Work that I've built out within workspaces I've configured includes emergency response validations, cause analysis, regulatory compliance checks, and a lot more. And that really is the beauty of working with AI and language models — we can essentially build anything we need per specification, simply through prompting and configuring our context.

Hopefully this is a clear differentiation between context and memories on one hand, and task modes — the specific instructions we set — on the other. As mentioned, we will be breaking down more of these concepts with real examples over the next week, and really starting to walk through different workflows and processes operating directly within Sierra.

As always, I really appreciate your time and attention. Until next time.
